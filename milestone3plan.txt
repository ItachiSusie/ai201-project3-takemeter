# Milestone 3 Plan (Collect + Annotate Dataset)

## Goal
Produce one final labeled CSV (single file, not pre-split) with at least **200** ChatGPT-Reddit examples from the Kaggle dataset, with no label above **70%**.

---

## Dataset source
- Source: `https://www.kaggle.com/datasets/armitaraz/chatgpt-reddit`
- Community domain: public Reddit discussions about ChatGPT
- Collection method: sample posts/comments from this dataset (not private/auth-required data)

---

## Label schema to apply
- `troubleshooting`
- `prompting_help`
- `news_policy_discussion`
- `showcase_story`

(Definitions are in `milestone2plan.txt` and should be treated as annotation rules.)

---

## File format and location
I will save one CSV in the repo, for example:
- `data/chatgpt_reddit_labeled.csv`

Minimum required columns:
1. `text` - full post/comment text used for classification
2. `label` - one string label from the 4-label taxonomy
3. `notes` - short justification for hard/ambiguous cases (can be empty for easy cases)

Optional helpful columns:
- `source_id` (original dataset id if available)
- `prelabeled_by_llm` (yes/no)
- `final_reviewed` (yes/no, should end as yes for all rows)

---

## Step-by-step annotation workflow
1. Load/sort Kaggle rows and remove obvious duplicates or empty text.
2. Build an annotation candidate pool >200 (target ~220) so I can drop low-quality rows.
3. Annotate row-by-row (no bulk skimming), applying the milestone 2 rules.
4. For ambiguous rows, write note: "considered X vs Y; chose Z because..."
5. If using LLM pre-labeling, manually review every pre-label before finalizing.
6. Keep only one final labeled CSV with at least 200 reviewed examples.

---

## Time plan (3-4 hours)
1. **0:00-0:30** - prepare candidate pool, clean duplicates/empty rows.
2. **0:30-2:30** - annotate first 150 rows carefully.
3. **2:30-3:30** - annotate remaining rows to reach >=200 and rebalance labels.
4. **3:30-4:00** - final QA pass + distribution check + edge-case notes update.

---

## Imbalance control plan
After first 200 labeled rows, count class distribution.

If any class >70%:
1. Pause finalization.
2. Add targeted rows for underrepresented classes from the same Kaggle dataset.
3. Continue annotation until no class exceeds 70%.

I will prefer a roughly even spread (about 50 per label) where possible.

## AI-assisted labeling policy (optional)
If I use AI pre-labeling:
1. I will pass exact label definitions from milestone 2.
2. I will require one label per row.
3. I will manually review every suggestion.
4. I will disclose AI pre-label use in the project AI-usage section.
